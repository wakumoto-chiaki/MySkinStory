## 1. サービス概要（※最後に書いてOK）
美容医療を定期的に受ける人が、肌の変化をストーリーとして残せるアルバムサービスである。

施術ごとの写真やメモを時系列で記録し、Before / Afterや振り返りを通して、自分の変化や努力を実感できる体験を提供する。

美容医療の記録管理ではなく、「肌の変化を楽しみながら残すこと」を目的としたサービスである。


## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情
美容皮膚科治療に携わってきた中で、お客様より、「施術を開始する前と今の肌状態を比較したい」「何年もこの施術を受けているけど実際に効果が出ているのか分からず通っている」「施術前に写真を撮っておこうと思うけどいつも忘れてしまう」といった声を多く聞いてきた。
その時に、お客様は自分の理想の肌になるために頑張って長い期間と高い費用を払って施術を継続してくださっているのに、目に見えた結果を得られないのは勿体無い！と感じていた。


### 2-2. なぜそれが「気になった」のか

私は、お客様のお悩みを改善したいという思いで日々一人一人のお客様と真摯に向き合い、施術をさせていただいていた。だからこそ、お客様にご満足いただける施術を提供したいという思いが強くあった。肌施術は基本的に数ヶ月から年単位で行っていくものであり、一回施術を受けたからといって効果実感を得られるものは少ない。だからこそ、何回も何年も治療を継続してくださったお客様には効果を目に見える形で実感していただきたいという思いが強くあった。


## 3. 課題の整理（表面的になっていないか）

### 3-1. 表に見えている困りごと

美容医療を受けているユーザーは、施術前後の写真を残したいと思っていても、撮影や記録を継続できないことが多い。また、過去の肌状態を振り返る機会が少ないため、自身の変化を実感しづらい。

### 3-2. 本当に解決したい課題は何か

美容医療の肌施術は数ヶ月から数年かけて継続して行うことが多いが、施術前の状態を振り返る機会が少なく、お客様自身が変化を実感しづらい。その結果、本来得られている変化や努力を実感できず、モチベーションの低下に繋がる。
私が解決したい課題は、「ユーザー自身が肌の変化を実感できないこと」である。

## 4. 想定ユーザーについて

### 4-1. 想定しているユーザー

美容医療を年数回以上受けている人
施術の変化を写真で残したいと思っているが、記録を継続できていない人

### 4-2. 自分とユーザーの距離

身近な誰かのために。
美容医療に以前携わっており、そこで出会ったお客様を想像しているため。



### 4-3. 実際に使われる可能性について(需要・存在確認)

美容医療の施術を定期的に受けているお客様は、様々な美容医療アプリのユーザーであることが多い。また、SNSに対する関心も高い傾向にある。そして、お客様自身も施術を受け始めた最初の方は写真を撮って記録をしようという意識は高い。だが、なかなか記録を継続できていないのが現状。そこで、単なる美容医療管理アプリではなく、美容医療の変化を残すライフログアルバムを作成することにより、ご自身の肌の状態の変化を何度も振り返り「綺麗になった自分」「ちゃんと続けた自分」を好きになれるこのアプリを利用したいと考える。
アプリのコンセプトは「肌の変化をストーリーとして残すアルバム」。



## 4-4. ユーザーがサービスを導入して利用するまでのイメージ(導入・継続の流れ)

美容医療を定期的に受けている、またはこれから受けようとしているユーザーが、施術の効果を記録したいと感じた時に利用を始めるサービス。ユーザーは新規登録後、「透明感プロジェクト」や「毛穴改善チャレンジ」などのストーリーを作成する。その後、施術後の写真やメモを定期的に記録していくことで、ストーリー詳細画面に変化の履歴が蓄積される。ユーザーはBefore / After比較や過去の記録を振り返ることで、自分の肌の変化や継続の軌跡を実感できる。最終的にストーリー詳細画面そのものが、「肌の変化をストーリーとして残すアルバム」となる。

- サービス名：instagram

- サービス名：TroveSkin
- URL：https://www.troveskin.com/
- どんなことができるか：肌写真の記録やbefore/After比較機能を提供するアプリ。


### 5-2. それでも自分が作りたい理由

美容医療管理アプリは記録がメインであり、実際の美容医療を受けているお客様も最初は登録するが、継続して利用することが難しいとおっしゃっていた。思わず定期的に更新したり開きたくなるアプリは何かなと考えた。


### 5-3. 差別化を「体験価値」で表現する

既存の美容医療アプリは、「施術を記録すること」が目的になっている。
本サービスでは、「変化を振り返ること」を目的としている。
施術単位ではなく、ストーリー単位で記録し、写真や気分や振り返りを蓄積することで、自分だけの美容アルバムを作れる体験を提供する。


## 6. このサービスで提供したい価値

### 6-1. ユーザーの変化

行動の変化：定期的にアプリを開き振り返りたくなる
気持ちの変化：自分の肌の変化のストーリーを見て、美容医療へのモチベーションが上がる
考え方の変化：写真を撮って記録することを面倒だなと感じるのではなく、写真を撮って自分のストーリーを完成させたいと考えるようになる


### 6-2. 価値を一文で表す

## 8. このアプリの懸念点とその対策


- 何が問題になりそうか：ユーザーが写真記録を継続できない可能性がある。
- なぜそう思うか：美容医療の肌施術における変化は数ヶ月から年単位で現れることが多く、日常的にアプリを開くモチベーションを維持しにくいため。
- そのために考えている対策：記録頻度を自由にする。ストーリー形式で振り返れるUIにする。Before / After比較により変化を実感しやすくする。節目ごとの振り返り機能を設ける。SNS共有機能により達成感を可視化する。

## 9. 今後の展開・発展の方向性


- 機能追加の方向性：自動アルバム生成。ストーリーサマリーカード生成。
- UI・体験の改善：年間振り返りページ。肌変化タイムラプス。
- 機能以外の工夫：美容医療クリニックとの連携

### 10-3.キャッチアップ不足の懸念

本アプリでは、ユーザーが肌の変化を振り返れる「ストーリーアルバム体験」の実現を最優先とする。当初はReactとRails APIモードを利用した構成を検討していたが、現時点ではキャッチアップ不足により実装難易度が高く、サービスの完成が遅れる可能性があると考えた。そのためMVPではRuby on Railsを用いてコア機能の実装を行い、まずはストーリー作成・記録投稿・Before / After比較といった最小限の価値提供を実現する。将来的には、SNS共有機能やReactの導入も検討している。まずは「美容医療管理アプリ」ではなく、「肌変化ストーリーアルバム」というコンセプトを体験できるサービスの完成を目指す。
